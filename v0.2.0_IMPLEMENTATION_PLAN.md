**Goal:** Enable optimization of ANY external API through plugin architecture

## 📝 Technical Breakdown

### **Phase 1: Core Interface & Architecture** 

#### Tasks:

**1. Design WorkloadSimulator Interface** (~4 hours)
```java
public interface WorkloadSimulator {
    // Execute load test with given parameters
    RunResult executeLoad(LoadTestConfig config);
    
    // Get simulator metadata
    SimulatorMetadata getMetadata();
    
    // Health check
    boolean isHealthy();
    
    // Cleanup resources
    void shutdown();
}

public class LoadTestConfig {
    private int concurrency;
    private int durationSeconds;
    private double targetRps;
    private Map<String, Object> customParams;
}

public class SimulatorMetadata {
    private String name;
    private String description;
    private String version;
    private List<String> requiredParams;
}
```

**2. Create Plugin Loading Mechanism** (~6 hours)
- ServiceLoader-based plugin discovery
- Configuration file parsing (YAML)
- Plugin validation
- Error handling

**3. Refactor RunnerMain for Plugins** (~4 hours)
- Remove hardcoded LoadRunner dependency
- Inject WorkloadSimulator via config
- Update orchestration logic
- Maintain backward compatibility

**4. Configuration System** (~6 hours)
```yaml
# application.yml
workload:
  simulator: http-rest  # or 'demo', 'database', etc.
  
http-rest:
  base-url: https://api.example.com
  endpoint: /v1/users
  method: GET
  headers:
    Authorization: Bearer ${API_TOKEN}
  timeout-ms: 5000
```

**Phase 1 Total: ~20 hours**

#### **Plugin 2: HTTP REST API Simulator** (~14 hours)

**What to build:**

**1. Core HTTP Client** (~6 hours)
- RestTemplate or WebClient configuration
- Request builder (method, headers, body)
- Response handling
- Error handling & retries
- Connection pooling

**2. Load Test Execution** (~4 hours)
- Multi-threaded request execution
- Latency measurement (same as existing)
- Throughput calculation
- Result aggregation

**3. Configuration Support** (~2 hours)
- Parse YAML config
- Environment variable substitution
- Validation

**4. Health Check** (~2 hours)
- Ping endpoint before test
- Validate connectivity
- SSL/TLS handling

**Example usage:**
```java
// User configuration
workload:
  simulator: http-rest

http-rest:
  base-url: https://my-api.com
  endpoint: /api/work
  method: POST
  body: '{"task": "process"}'
  headers:
    Content-Type: application/json
    X-API-Key: ${API_KEY}
```

**Effort:** MEDIUM (new code, but straightforward HTTP client work)
