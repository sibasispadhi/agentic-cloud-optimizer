## ✅ Core Functionality

### **1. Self-Contained Load Testing & Optimization Demo**

**What it does:**
- Runs a built-in demo API endpoint (`/v1/work`) that simulates real microservice work
- Measures baseline performance (latency, throughput, concurrency)
- Uses AI agents to analyze metrics and decide optimizations
- Applies the optimization and re-measures performance
- Generates before/after comparison reports

**What it is:**
- ✅ A working proof-of-concept demonstration
- ✅ Runs completely offline (no cloud APIs required)
- ✅ Produces real, measurable optimization results

**What it is NOT:**
- ❌ A plugin for external APIs (yet)
- ❌ Production-ready for live services
- ❌ Multi-service orchestration

---

## 🤖 Dual Agent System

### **Agent 1: SimpleAgent (Rule-Based)**

**What it does:**
- Analyzes median latency from metrics
- Applies deterministic threshold-based rules:
  - If latency > target (100ms) → INCREASE concurrency
  - If latency < 70% of target → DECREASE concurrency  
  - Otherwise → MAINTAIN current setting
- Generates explainable decision with reasoning
- 95% confidence score (deterministic)

**Input:** 20 recent latency metrics + current concurrency  
**Output:** Concurrency recommendation (e.g., "Set concurrency to 6")  
**Reasoning:** Plain English explanation (e.g., "Latency exceeds target, increasing concurrency")  

**Use case:** Fast, predictable, no dependencies

---

### **Agent 2: SpringAiLlmAgent (AI-Powered)**

**What it does:**
- Uses local LLM (Ollama + Llama2) for intelligent analysis
- Considers multiple factors simultaneously
- Generates context-aware optimization recommendations
- Provides detailed natural language reasoning
- 100% offline (no API keys, no cloud calls)

**Input:** Metrics + current concurrency + target latency  
**Output:** JSON decision with newConcurrency, expectedLatency, explanation  
**Reasoning:** AI-generated natural language explanation  

**Requirements:** Ollama installed + Llama2 model pulled  
**Use case:** Intelligent, context-aware, educational

## 📊 Metrics & Measurement

### **Load Testing Capabilities**

**What you can configure:**
- Concurrency level (number of parallel threads)
- Test duration (seconds)
- Target RPS (requests per second, 0 = unlimited)

**What it measures:**
- **Latency:**
  - Median (p50)
  - Average (mean)
  - 95th percentile (p95)
- **Throughput:**
  - Requests per second (RPS)
  - Total requests completed
- **Cost:**
  - Estimated cost based on request count

**Metrics logging:**
- Real-time async logging to JSONL files
- In-memory buffer for recent metrics
- Timestamped metric events

## ⚙️ Configuration Options

### **Agent Selection**
```bash
# Rule-based agent (default)
java -Dagent.strategy=simple -jar agent-cloud-optimizer-0.1.0.jar

# AI-powered agent (requires Ollama)
java -Dagent.strategy=llm -jar agent-cloud-optimizer-0.1.0.jar
```

### **Load Test Parameters**
```bash
java -Dbaseline.concurrency=8 \
     -Dload.duration=15 \
     -Dtarget.rps=100 \
     -jar agent-cloud-optimizer-0.1.0.jar
```

### **Via Environment Variables**
```bash
export AGENT_STRATEGY=simple
export CONCURRENCY=4
export DURATION=10
export TARGET_RPS=0
./scripts/run-agent.sh
```

### **Via Configuration File**
`src/main/resources/application.yml`
```yaml
agent:
  target-latency-ms: 100.0
  strategy: simple

baseline:
  concurrency: 4
  
load:
  duration: 10
  target-rps: 0
```
